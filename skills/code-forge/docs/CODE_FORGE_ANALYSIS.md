# Code-Forge 技能集逻辑与架构分析报告

这份名为 **Code-Forge** 的技能集（Skills）是一个高度结构化、模块化且面向 **软件开发生命周期（SDLC）** 全流程的 AI 助手增强框架。它旨在将复杂的软件开发任务拆解为可预测、可验证且标准化的原子操作。

---

## 一、 核心设计思路：软件工程即流程（SDLC as a Workflow）

Code-Forge 的核心思想是**将 AI 从一个“简单的聊天机器人”转变为一个“遵循严谨工程规范的虚拟程序员”**。它不仅仅是代码生成，而是覆盖了从需求分析到交付的每一个环节。

1.  **阶段化（Phasing）**：通过 `plan` (规划), `impl` (实现), `verify` (验证), `review` (评审) 等指令，强制 AI 遵循“先想后做，做完自检”的工程逻辑。
2.  **状态驱动（State-Driven）**：在 `examples` 中可以看到 `state.json` 和 `tasks/` 目录，这表明该系统能够维护开发进度。AI 不再是健忘的，而是通过文档 and 状态文件记录当前执行到哪一步。
3.  **标准化（Standardization）**：通过 `templates/` 和 `shared/` 下的规范（如 `coding-standards.md`），确保无论开发什么项目，生成的代码风格、文档结构和目录组织都是高度一致的。

---

## 二、 目录逻辑与功能拆解

*   **`commands/` (交互入口)**：定义了用户可以调用的指令集。
    *   `plan.md`, `impl.md`, `verify.md`：构成了核心开发循环。
    *   `parallel.md`：可能用于并行处理任务，提升效率。
    *   `worktree.md`：暗示了对 Git 工作流的深度集成，支持在独立环境下开发。
*   **`skills/` (专家角色定义)**：这是系统的“大脑”。
    *   每个子目录下的 `SKILL.md` 包含了该特定任务（如 `debug` 或 `tdd`）的深度领域知识和操作规程。
    *   `shared/` 目录下的配置和标准是所有技能的基石，避免了规则重复，保证了全局统一。
*   **`docs/` (治理与配置)**：
    *   提供了框架本身的配置层级（`CONFIG_HIERARCHY.md`）和目录设计规范，确保开发者知道如何扩展和维护这套技能集。
*   **`examples/` (参考实现)**：
    *   通过 `user-auth` 示例，展示了从输入需求到生成任务列表、状态记录和最终输出的全过程，作为 AI 学习和人类参考的基准。

---

## 三、 为什么这么写？（深度动机）

1.  **解决“上下文膨胀”问题**：
    *   如果让 AI 一次性完成所有任务，上下文会迅速耗尽且逻辑易混乱。通过 `plan` -> `impl` -> `verify` 的拆解，每次任务只关注当前阶段，极大的提高了 AI 的准确性和逻辑深度。
2.  **提升可靠性与可测试性**：
    *   引入 `tdd.md`（测试驱动开发）和 `verify.md`，意味着每一行代码的产生都伴随着验证机制。这降低了 AI 产生“幻觉”代码的风险。
3.  **支持复杂工程**：
    *   普通的 AI 助手难以处理大型项目。Code-Forge 通过 `DIRECTORY_DESIGN.md` 和 `FILE_STRUCTURE.md` 定义了标准，使 AI 能够理解并维护复杂的文件系统结构。
4.  **易于协作（Human-in-the-loop）**：
    *   生成的 `plan.md` 和 `tasks/` 是人类可读的。人类可以在 `review` 阶段介入，修正 AI 的偏差，然后再让 AI 继续执行 `impl`。

---

## 四、 预期达到的结果

1.  **工程质量的飞跃**：输出的代码不再是散乱的代码片段，而是包含测试、文档、符合架构规范的完整工程。
2.  **极高的开发效率**：人类只需输入高层级需求（`input/user-auth.md`），AI 就能自动拆解为原子任务并逐步实现，减少了繁琐的脚手架搭建和基础逻辑编写时间。
3.  **知识沉淀**：通过 `shared/coding-standards.md`，团队的编程经验被固化在技能集中，新加入的 AI 或人类成员都能立即遵循同样的标准。
4.  **全自动/半自动交付**：最终能够实现从 `forge`（锻造/发起）到 `finish`（完成/清理）的闭环，达成“软件工厂”式的自动化生产。
